Input

The Input component provides a customizable text input field with built-in validation and optional floating labels.

Features​

  • Customizable Validation: Supports multiple validation rules including minLength, maxLength, email, and numeric validation.
  • Floating Labels: Option to display the label as a floating placeholder when the input is focused or has a value.
  • Real-time Validation: Validates the input on change and blur events, displaying error messages when validation fails.
  • Customizable Placeholder and Label: Easily customizable placeholder text and labels.
  • Required Field Handling: Marks fields as required with proper validation feedback.

Usage​

Import the Input component into your React file

import { Input } from "ghewar";

Then, you can use the <Input /> component anywhere in your JSX code

Example​

This is a basic example how can we create an input field to handle the emails.

  • Code

Email address:

Form Valid Status: true

import { useState } from "react";
import { Input } from "ghewar";

const InputDemo = () => {
const [email, setEmail] = useState<string>("");
const [isFormValid, setIsFormValid] = useState<boolean>(true);

const handleEmailChange = (e: React.ChangeEvent<HTMLInputElement>) => {
  setEmail(e.target.value);
};

const handleValidityChange = (isValid: boolean) => {
  setIsFormValid(isValid);
};

return (
  <>
    <h2>Email address: {email}</h2>
    <h2>Form Valid Status: {`${isFormValid}`}</h2>
    <div>
      <Input
        id="email"
        placeholder="Enter your email"
        onChange={handleEmailChange}
        validator={{ email: true }}
        onValidityChange={handleValidityChange}
      />
    </div>
  </>
);
};

export default InputDemo;

Validators​

You can pass various validators to the input component based on your requirements.

  • email: Validates that the input is a properly formatted email address.
  • minLength: Ensures the input meets the specified minimum character length.
  • maxLength: Limits the input to the specified maximum number of characters.
  • numeric: Ensures the input contains only numeric values.

Floating Label​

If we set the floating label then the input will change the behaviour.

  • Preview
  • Code
import { useState } from "react";
import { Input } from "ghewar";

const InputDemo = () => {
const [name, setName] = useState<string>("");

const handleNameChange = (e: React.ChangeEvent<HTMLInputElement>) => {
  setName(e.target.value);
};

return (
  <>
    <div style={{ width: "400px" }}>
      <Input
        id="name"
        placeholder="Enter your name"
        label="Name"
        floatinglabel={true}
        onChange={handleNameChange}
        required
      />
    </div>
  </>
);
};

export default InputDemo;

Props​

PropTypeDescription
placeholderstringThe placeholder text displayed inside the input field.
requiredbooleanIndicates if the input field is required.
labelstringThe label text for the input field. This is mandatory if we set the floating label to true.
validatorValidatorAn object containing validation rules (e.g., email, minLength, maxLength, numeric).
floatinglabelbooleanIf true, the label will be a floating label that animates above the input field when focused.
onValidityChange(isValid: boolean) => voidA callback function that is triggered whenever the input field's validity state changes.
onChange(e: ChangeEvent<HTMLInputElement>) => voidA callback function that is called whenever the input value changes.
idstringThe unique identifier for the input element.
typestringThe type of input (e.g., "text", "email", "password").
